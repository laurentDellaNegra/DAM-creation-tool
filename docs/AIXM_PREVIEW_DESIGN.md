# AIXM Preview Side Panel

## Goal

Add a persistent right-side AIXM preview workspace before Send. Operators can
inspect the generated AIXM, explicitly enter XML edit mode, save supported XML
changes back into the left form, then send or download the synchronized payload.

## Source of Truth

The editable DAM form and `dam-core` domain model remain authoritative. The UI
never sends an arbitrary XML editor buffer. XML changes are accepted only by
parsing supported AIXM fields into a candidate domain model, validating that
candidate, and regenerating canonical AIXM from it.

Unsupported XML content is not preserved as opaque state. Later file import will
reuse the same parser/mapper, extract supported fields, and warn about dropped
or unsupported content.

## Panel Behavior

- The panel is toggled from the toolbar and docks as a resizable right panel.
- When clean, the XML preview live-regenerates from the left form.
- If the form cannot generate valid supported AIXM, the panel shows blocking
  validation/export errors instead of editable XML.
- The default panel state is read-only canonical preview.
- `Edit XML` unlocks raw editing and creates a dirty editor buffer.
- While dirty, left-form changes do not overwrite the XML draft.
- `Save XML` is the only point where XML changes update the left form.
- `Discard XML changes` restores the latest canonical form-generated XML.
- Closing a dirty panel asks for confirmation before discarding changes.
- Global Send, panel Send, and Download AIXM are blocked while XML edits are
  dirty; the user must save or discard first.

## Supported Round-Trip Fields

The first implementation only maps fields currently emitted by the AIXM exporter:

- predefined `ext:mapId` and catalog-backed `aixm:name`;
- manual polygon `aixm:name`, `gml:posList`, and label `gml:pos`;
- validity start/end dates;
- one activation period start/end times;
- start/end display flags;
- lower/upper levels and units;
- `ext:displayLevels`.

The mapper preserves form-only fields from the current left panel, including
distribution, free text, altitude correction, upper/lower buffer selections, and
unsupported manual-map attributes.

## Rejection Rules

- Predefined map geometry and label edits are rejected.
- Predefined `ext:mapId` must resolve to a bundled catalog map.
- Predefined `aixm:name` must match the catalog map name.
- Manual maps must keep `ext:mapId` equal to `0`.
- Manual geometry round-trip is supported only for manual polygon mode.
- Multiple AIXM activation periods are rejected until export supports them.
- `gml:beginPosition` supplies the full start date/time.
- `aixm:startDate` must match the begin-position day/month.
- `aixm:endDate` infers its year from the start date, rolling forward only when
  needed for a non-decreasing date range.
- `gml:beginPosition` time and `aixm:startTime` must agree.

The parser accepts semantic equivalents rather than exact generated bytes: it
does not depend on generated `gml:id` values, whitespace, element order, or
harmless namespace prefix choices.
